# 📱 GitHub 手机端博客管理指南

## 🚀 快速开始

### 1. 准备工作
- 在手机上安装 GitHub App
- 确保你有仓库的写入权限
- 了解基本的 Markdown 语法

### 2. 文章发布流程

#### 方式一：直接在 GitHub App 中编辑
1. 打开 GitHub App，进入你的博客仓库
2. 导航到 `src/content/posts/` 目录
3. 点击右上角 `+` 创建新文件
4. 复制下面的文章模板
5. 填写内容后提交更改
6. 等待 GitHub Actions 自动部署（约 2-3 分钟）

#### 方式二：使用 Markdown 编辑器
1. 在手机 Markdown 编辑器中编辑文章
2. 复制文章内容
3. 在 GitHub App 中创建文件并粘贴
4. 提交更改

## 📝 文章模板

```markdown
---
title: "文章标题"
date: "2026-06-11"
updated: "2026-06-11"
summary: "文章摘要，用于首页展示"
tags:
  - 标签1
  - 标签2
category: "分类名称"
draft: false
---

# 文章内容

在这里开始写你的文章...

## 二级标题

正文内容...

### 三级标题

更多内容...
```

## 🎯 Front Matter 字段说明

| 字段 | 说明 | 必填 | 示例 |
|------|------|------|------|
| `title` | 文章标题 | ✅ | `"我的第一篇文章"` |
| `date` | 发布日期 | ✅ | `"2026-06-11"` |
| `updated` | 更新日期 | ❌ | `"2026-06-12"` |
| `summary` | 文章摘要 | ✅ | `"这是一篇关于..."` |
| `tags` | 标签数组 | ✅ | `["技术", "生活"]` |
| `category` | 文章分类 | ❌ | `"技术笔记"` |
| `draft` | 是否草稿 | ❌ | `false` |

## 📂 文件命名规范

- 使用英文文件名，避免中文
- 使用连字符 `-` 分隔单词
- 文件名要能反映文章内容
- 示例：`my-first-post.md`、`tech-guide.md`

## 🔄 自动部署流程

1. 你在手机上提交文章到 GitHub
2. GitHub Actions 自动触发构建
3. 构建完成后自动部署到 GitHub Pages
4. 约 2-3 分钟后即可在网站上看到新文章

## 💡 手机端操作技巧

### 1. 快速创建文章
- 复制文章模板到剪贴板
- 在 GitHub App 中创建新文件
- 粘贴模板并修改内容

### 2. 预览文章
- 使用 Markdown 编辑器的预览功能
- 或在电脑上查看本地预览

### 3. 图片处理
- 将图片放到 `public/assets/` 目录
- 在文章中使用相对路径引用：`![图片描述](/assets/图片名.webp)`

### 4. 批量操作
- 可以一次性创建多篇文章
- 每篇文章单独提交，便于追踪

## 🛠️ 常见问题

### Q: 如何修改已发布的文章？
A: 在 GitHub App 中找到对应的 `.md` 文件，点击编辑，修改后提交即可。

### Q: 如何删除文章？
A: 在 GitHub App 中找到对应的 `.md` 文件，点击删除，提交更改。

### Q: 如何设置文章为草稿？
A: 在 front matter 中设置 `draft: true`，文章将不会在网站上显示。

### Q: 部署失败怎么办？
A: 检查 GitHub Actions 的运行日志，常见原因是 front matter 格式错误。

### Q: 如何查看部署状态？
A: 在 GitHub 仓库的 "Actions" 标签页查看部署进度和结果。

## 📞 获取帮助

如果遇到问题：
1. 检查 GitHub Actions 日志
2. 确认 front matter 格式正确
3. 查看项目 README.md 了解更多信息
